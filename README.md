# IMPACT Magazine Website

## 프로젝트 개요
**IMPACT 매거진**은 사회적 이슈와 기술 변화를 분석해 일상의 문제를 발견하고 그 본질을 탐구하는 디지털 매거진 플랫폼입니다.

### 프로젝트 목표
- 청년 창업가 인터뷰와 사회혁신 사례를 통한 인사이트 제공
- 반응형 웹 디자인으로 모바일과 데스크톱에서 최적화된 사용자 경험 제공
- 직관적인 콘텐츠 네비게이션과 몰입형 읽기 경험 구현

## 현재 구현된 기능들

### ✅ 핵심 기능
1. **메인 홈 페이지**
   - 매거진 스타일의 그리드 레이아웃
   - 카테고리별 콘텐츠 구성
   - 페이지네이션 지원
   - 반응형 디자인 (모바일/데스크톱)

2. **게시물 뷰어**
   - 풀스크린 오버레이 형태의 게시물 표시
   - 읽기 진행률 표시 (Progress Bar)
   - 이미지 모달 뷰어
   - 관련 게시물 추천
   - 에디터 프로필 연결

3. **카테고리 시스템**
   - TECH, EAT, STYLE, CULTURE, LIFE, EDITOR'S PICK
   - 카테고리별 필터링
   - 카테고리별 페이지네이션

4. **에디터 페이지**
   - 에디터 목록 및 프로필
   - 에디터별 작성 글 모아보기
   - 에디터 상세 정보 표시

5. **검색 기능**
   - 실시간 검색
   - 검색 결과 즉시 표시
   - 글로벌 네비게이션 검색 지원

6. **ABOUT 페이지**
   - 매거진 소개
   - 2025-2029 로드맵
   - 브랜드 스토리

### ✅ 최근 구현된 개선사항 (2025-11-02)

1. **페이지 진입 시 스크롤 위치 초기화**
   - 모든 페이지/게시물 진입 시 자동으로 최상단부터 표시
   - 부드러운 전환 효과 제거로 즉각적인 페이지 로드

2. **독립적 스크롤 관리**
   - ABOUT, 게시물, EDITORS 페이지에서 메인 홈 스크롤 중단
   - 각 오버레이 페이지별 독립적인 스크롤 영역
   - 모바일과 데스크톱 모두에서 일관된 동작

3. **자동 콘텐츠 로딩**
   - 서브 페이지에서 새로고침 후 메인 홈 복귀 시 자동 로딩
   - 백그라운드 프리로딩으로 빠른 페이지 전환
   - 로딩 상태 표시로 사용자 경험 개선
4. **개선된 페이지네이션 시스템**
   - 메인 홈페이지 하단 중앙에 페이지 네비게이션 추가
   - 한 페이지당 20개 게시물 표시
   - 현재 페이지 번호 주황색(#ff6b35) 하이라이트
   - 첫 페이지에서는 `<<` 버튼 미표시
   - 마지막 페이지에서는 `>>` 버튼 미표시
   - 페이지 번호 표시: `<< 1 2 3 ... n >>`
   - 다크모드 완벽 지원 (모든 숫자가 명확하게 표시)
   - 반응형 디자인 (모바일: 5개 페이지 표시, 데스크탑: 10개 페이지 표시)

## 기술 스택
- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **API**: WordPress REST API (Netlify Functions 프록시)
- **호스팅**: Netlify
- **분석**: Google Analytics (G-D0DVTWYNGN)

## URL 구조

### 메인 페이지
- `/` - 홈페이지

### 파라미터 기반 라우팅
- `/?post={id}` - 게시물 상세 보기
- `/?category={slug}` - 카테고리별 게시물
- `/?page=about` - ABOUT 페이지
- `/?page=editors` - 에디터 목록
- `/?editor={id}` - 특정 에디터 프로필
- `/?page=search` - 검색 페이지

### 해시 기반 라우팅 (레거시 지원)
- `/#post-{id}` - 게시물 직접 링크
- `/#editor-{id}` - 에디터 직접 링크
- `/#about` - ABOUT 페이지
- `/#editors` - 에디터 목록
- `/#search` - 검색 페이지

## 데이터 구조

### 게시물 (Posts)
```javascript
{
  id: Number,
  title: { rendered: String },
  content: { rendered: String },
  excerpt: { rendered: String },
  date: String,
  _embedded: {
    author: Array,
    'wp:featuredmedia': Array,
    'wp:term': Array
  }
}
```

### 에디터 (Users)
```javascript
{
  id: Number,
  name: String,
  slug: String,
  description: String,
  avatar_urls: Object
}
```

### 카테고리
- `tech` (ID: 3) - 기술과 혁신
- `eat` (ID: 5) - 음식과 외식 문화
- `style` (ID: 6) - 패션과 라이프스타일
- `culture` (ID: 7) - 문화와 예술
- `life` (ID: 8) - 일상과 삶
- `editors-pick` (ID: 4) - 에디터 추천

## 개발 가이드

### 로컬 테스트
1. 프로젝트 파일을 웹 서버에서 실행
2. API 프록시가 설정된 환경에서 테스트
3. 반응형 디자인 확인 (모바일/태블릿/데스크톱)

### 배포
1. Publish 탭으로 이동
2. "Publish" 버튼 클릭
3. 자동으로 Netlify에 배포됨
